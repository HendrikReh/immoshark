# Stakeholder-Dokumentation

Rollenspezifische Dokumente für alle Projektbeteiligten. Jedes Dokument ist auf die Informationsbedürfnisse der jeweiligen Zielgruppe zugeschnitten.

## Übersicht

| Dokument | Zielgruppe | Inhalt |
|----------|------------|--------|
| [Benutzeranleitung](enduser/anleitung.md) | Endanwender | Bedienung mit Screenshots: Dashboard, Liste, Detail, Formular, CSV-Import |
| [Projektmanager](projektmanager.md) | Projektmanagement | Geschäftswert, Projektstatus, Risikobewertung, Meilensteine |
| [Backend-Entwickler](backend-entwickler.md) | Backend-Entwicklung | Architektur, DB-Patterns, Service-Layer, Middleware, Konventionen |
| [Frontend-Entwickler](frontend-entwickler.md) | Frontend-Entwicklung | Komponenten, Hooks, API-Client, Vite-Config, Shared Types |
| [Tester / QA](tester.md) | Testing | Test-Suiten, Helfer, manuelle Testszenarien, Edge Cases |
| [Ops / DevOps](ops-devops.md) | Betrieb / Deployment | Systemanforderungen, Prozess-Management, Backup, Monitoring, Troubleshooting |

## Wann welches Dokument lesen?

- **Ich will die App benutzen** → [Benutzeranleitung](enduser/anleitung.md)
- **Ich will den Projektstatus verstehen** → [Projektmanager](projektmanager.md)
- **Ich will am Backend arbeiten** → [Backend-Entwickler](backend-entwickler.md)
- **Ich will am Frontend arbeiten** → [Frontend-Entwickler](frontend-entwickler.md)
- **Ich will Tests schreiben oder ausführen** → [Tester / QA](tester.md)
- **Ich will die App deployen oder betreiben** → [Ops / DevOps](ops-devops.md)
