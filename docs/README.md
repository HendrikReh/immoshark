# docs/

Gesamte Projektdokumentation für ImmoShark.

## Struktur

```text
docs/
├── assets/                     Bilder für die README (Dashboard-Screenshot)
└── stakeholder/                Rollenspezifische Dokumentation
    ├── README.md               Übersicht aller Stakeholder-Dokumente
    ├── enduser/                Benutzeranleitung mit Screenshots
    ├── projektmanager.md       Geschäftswert, Status, Risiken
    ├── backend-entwickler.md   Architektur, DB, Services, Middleware
    ├── frontend-entwickler.md  Komponenten, Hooks, API-Client, Styling
    ├── tester.md               Test-Suiten, Helfer, manuelle Szenarien
    └── ops-devops.md           Deployment, Backup, Monitoring
```

## Schnelleinstieg

| Ich will ... | Dokument |
|---|---|
| die App benutzen | [Benutzeranleitung](stakeholder/enduser/anleitung.md) |
| den Projektstatus verstehen | [Projektmanager](stakeholder/projektmanager.md) |
| am Backend arbeiten | [Backend-Entwickler](stakeholder/backend-entwickler.md) |
| am Frontend arbeiten | [Frontend-Entwickler](stakeholder/frontend-entwickler.md) |
| Tests schreiben oder ausführen | [Tester / QA](stakeholder/tester.md) |
| die App deployen oder betreiben | [Ops / DevOps](stakeholder/ops-devops.md) |

## Assets
